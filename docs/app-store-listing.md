# App Store 商店页文案与发布计划

> 定位方法:April Dunford(竞争框架/取舍)+ Bob Moesta(struggling moment)。
> 发布方法:soft/hard launch 分离 + 同心圆扩散 + lightning strike。

## 一、定位(所有文案的地基)

- **目标用户**:用 Mac 办公的个人与 3-20 人小团队负责人;要和 Word/Excel 用户交换文件;受够了广告弹窗和会员套娃
- **切换来源(竞争框架)**:WPS(免费但广告+会员拆功能)、M365(¥398/年,重)、Pages/Numbers(发给 Windows 同事格式就乱)、腾讯文档/石墨(格式兼容弱)
- **Struggling moment**:打开办公软件先关三个弹窗;导出 PDF 提示开会员;"最终版v3(2).docx"满天飞
- **差异化(只有我们能说的真话)**:免费版就是完整版(AGPL 开源背书)· 原生 docx/xlsx/pptx · 自动版本历史 · AI 内置 · 数据处理透明可自部署
- **明确不做(取舍)**:重度宏/VBA 用户;纯离线场景

## 四、发布计划(零预算,soft/hard 分离)

**Phase 0 · Soft launch(过审即上架,不吆喝,1-2 周)**
- 目标:跑通真实内购、收集 10 个真实用户反馈、修致命问题
- 同心圆最内圈:自己 → 朋友/前同事,人肉发安装链接
- 期间录好功能视频、攒 2-3 条真实用户好评(评分引导放在"保存成功"等愉悦时刻)

**Phase 1 · Hard launch(选一周集中打,lightning strike)**
- 中文线(主):V2EX 创造者板块(角度:「一个人把开源云 Office 做上了 Mac App Store」开发者故事)→ 少数派 Matrix 投稿 → 即刻/小红书 Mac 生产力话题
- 英文线(副):Hacker News「Show HN」+ r/selfhosted(角度:AGPL 开源、可自部署;README 已双语)
- 同一周内全部发出,互相导流制造动量;每个帖子回复前 48 小时全程在线
- PR 目的是可信度不是拉新:拿到的报道/热帖截图回填商店页和官网「媒体报道」

**Phase 2 · 节奏化(每 4-6 周一个"小发布")**
- 候选弹药:模板库上线 / AI 新动作 / Windows 版 / 团队空间。每次都按小型 launch 走一遍社区

**验证钩子(接 measuring-product-market-fit)**
- Hard launch 后 3 个月看自然注册留存;若无自然增长,回到 PMF 诊断而不是加大投放
